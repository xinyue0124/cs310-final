─────────────────────────────────────────────────────────────────────
  PART 1 — INTRODUCTION  (about 1 minute)
  [Face camera or show title slide]
─────────────────────────────────────────────────────────────────────

"Hi, my name is Xinyue Zhang, and this is my CS 310 final project.

The app is called Animal Rescue Hub. I built it because I actually
adopted my cat from a small rescue called Paws and Claws in Evanston,
and I noticed that small rescues often don't have great tools for
getting visibility online or managing adoption inquiries.

So the idea is — a rescue staff member can upload a photo of an animal,
the server automatically figures out what species it is using AWS
Rekognition, saves it to the database, and emails all subscribers to
let them know a new animal is available. Visitors can browse, and when
they find an animal they want to adopt, they submit an application and
get a confirmation email back.

There are three non-trivial server-side operations, which I'll walk
through one by one. Let me start by showing you the architecture."


─────────────────────────────────────────────────────────────────────
  PART 2 — ARCHITECTURE  (about 1.5 minutes)
  [Show the architecture diagram — architecture-diagram.png]
─────────────────────────────────────────────────────────────────────

"So here's how the system is put together.

On the left is the client — I have a React single-page app that runs
in the browser, and also a Python command-line client for testing.
Both of them talk to the server over HTTP with JSON.

In the middle is the Node.js Express web service, running on AWS
Elastic Beanstalk. This is the main focus of the project — it handles
all the logic and connects everything together.

On the right are the backend services. The server talks to:
  — Amazon S3 to store and retrieve pet photos
  — Amazon RDS, which is a MySQL database, to store pet listings,
    subscribers, and adoption applications
  — AWS Rekognition to automatically detect what animal is in a photo
  — Amazon SES to send email notifications
  — and The Cat API, which is a free external API that gives us
    live cat breed information

The dashed border on The Cat API means it's external — not part of AWS.

Alright, let me open the app and show you how it works."

"The first tab is called Cat Breeds. Every time this page loads, the

This is Non-Trivial Operation number one. The server calls the
external API, transforms the response into our own schema, and sends
it back to the browser.

You can see each breed has a photo, a description, lifespan, weight,

─────────────────────────────────────────────────────────────────────
  PART 4 — WRAP UP  (about 30 seconds)
─────────────────────────────────────────────────────────────────────

"So to summarize the three non-trivial operations:

Number one — GET /external_pets — the server calls The Cat API
live and returns real breed data.

Number two — POST /listing — uploads a photo to S3, runs
Rekognition to auto-detect the species, saves to MySQL, and
emails all subscribers via SES.

Number three — POST /apply — saves an adoption application to
the database and sends a personalized SES confirmation email.

Everything runs on Node.js with Amazon RDS, S3, Rekognition,
SES, and The Cat API on AWS.

Thanks for watching!"
